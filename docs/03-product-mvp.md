# MVP Definition (Moat-First)

The MVP exists to create switching pain via payroll dependency.

## Core Principle

No time can be tracked without a job.

## Roles

### Owner / Admin
- Creates jobs
- Exports payroll
- Reviews labour cost per job

### Worker
- Clocks in/out
- Selects job
- No account creation

## MVP Features

1. Mandatory Job Selection
- Workers must select a job to clock in
- No generic shifts

2. Magic Link Worker Access
- SMS-based magic link
- Web app only
- No passwords

3. Job Creation
- Manual creation
- CSV upload

4. Timesheets
- Worker, job, hours
- Simple table view

5. Payroll Export
- CSV format
- Includes job attribution and labour cost

6. Job Labour Summary
- Labour hours and cost per job
- Core retention screen

7. Weekly Labour Email
- Automatic weekly summary
- Reinforces habit

8. Payroll Leak Detection (Basic)
- Overlapping shifts
- Excessively long shifts
- Missing clock-outs

## Explicit Exclusions
- No scheduling
- No chat
- No GPS (Phase 1.5)
- No mobile apps
